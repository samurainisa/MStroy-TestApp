# MStroy-TestApp

Для наглядности добавлю рисунок изначального графа в задании, чтоб понятнее было писать тесты
![graph](graph.png)

изначально я решал задачу через массивы, сложности была примерно 0(n^2), если прогонять тесткейсы с большими данными

потом после переписал код с использованием Map, сложность стала O(n)

1. написан класс для работы с графом TreeStore (индексы на основе Map, без O(n) перебора на каждый вызов)
2. написаны методы 
   2.1. getAll() - возвращает все элементы графа, сложность O(n) (только на копирование Map в массив)
   2.2. getItem(id: TreeItemId) - возвращает элемент графа по id, сложность O(1)
   2.3. getChildren(id: TreeItemId) - возвращает дочерние элементы графа по id, сложность O(1)
   2.4. getAllChildren(id: TreeItemId) - возвращает все дочерние элементы графа по id, сложность O(k), где k - количество потомков
   2.5. getAllParents(id: TreeItemId) - возвращает всех родителей элемента, сложность O(h), где h - глубина дерева
   2.6. addItem/updateItem/removeItem - добавление, обновление и удаление (вместе с потомками) элемента, сложность O(1) для addItem/updateItem и O(k) для removeItem
3. написаны юнит-тесты для класса TreeStore 
4. проект разбит на модули (компоненты, тесты, стор, utils)
5. проект написан на TypeScript + Vite + Vitest + ESLint + Prettier

## установка зависимостей проекта

```sh
npm install
```
